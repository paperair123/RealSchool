# RealSchool模组开发文档

## 项目概述

RealSchool（真实学校）是一个基于Minecraft 1.21.1和Fabric模组加载器的校园主题模组。该模组旨在为Minecraft添加丰富的校园生活元素，包括教学设施、食物系统、矿物资源、机器设备等，让玩家能够在游戏中体验真实的校园生活。

### 基本信息
- **模组名称**: RealSchool (真实学校)
- **版本**: 1.0.0
- **Minecraft版本**: 1.21.1
- **模组加载器**: Fabric
- **包名**: papercliper.realschool

## 核心类详解

### 1. 主模组类 (RealSchool.java)

这是模组的入口点，实现了`ModInitializer`接口。主要功能包括：

- **初始化顺序**: 按照依赖关系依次初始化各个模块
- **日志记录**: 使用SLF4J记录模组加载过程
- **模块注册**: 调用各个子模块的注册方法

```java
@Override
public void onInitialize() {
    ModItems.registerModItems();           // 物品注册
    ModItemGroups.registerModItemGroups(); // 物品栏注册
    ModBlocks.registerModBlocks();         // 方块注册
    ModEntities.registerModEntities();     // 实体注册
    ModWorldGeneration.generateModWorldGen(); // 世界生成注册
}
```

### 2. 客户端初始化 (RealSchoolClient.java)

负责客户端特有的初始化工作，如实体渲染器注册：

- **实体渲染器**: 注册学生实体的渲染器
- **客户端资源**: 处理客户端特有的资源加载

### 3. 数据生成器 (RealSchoolDataGenerator.java)

实现了`DataGeneratorEntrypoint`接口，负责生成模组的数据文件：

- **语言文件**: 中文和英文翻译
- **配方文件**: 合成、烧炼等配方
- **模型文件**: 物品和方块的模型
- **标签文件**: 方块和物品的标签
- **战利品表**: 方块破坏掉落物
- **世界生成数据**: 矿石生成配置

public static Block register(String id, Block block) {

## 世界生成系统

世界生成系统负责在Minecraft世界中自然生成模组的矿石。该系统包含四个核心文件：

### ModConfiguredFeatures.java

定义矿石生成的基本配置，包括矿脉大小和替换规则：

```java
public static void bootstrap(Registerable<ConfiguredFeature<?, ?>> context) {
    RuleTest stoneReplaceables = new TagMatchRuleTest(BlockTags.STONE_ORE_REPLACEABLES);
    RuleTest deepslateReplaceables = new TagMatchRuleTest(BlockTags.DEEPSLATE_ORE_REPLACEABLES);
    
    // 石膏矿石配置 - 矿脉大小9
    List<OreFeatureConfig.Target> gypsumOreTargets = List.of(
        OreFeatureConfig.createTarget(stoneReplaceables, ModBlocks.GYPSUM_ORE.getDefaultState()),
        OreFeatureConfig.createTarget(deepslateReplaceables, ModBlocks.GYPSUM_ORE.getDefaultState())
    );
    
    register(context, GYPSUM_ORE_KEY, Feature.ORE, new OreFeatureConfig(gypsumOreTargets, 9));
}
```

**配置的矿石**：
- **石膏矿石**: 矿脉大小9，可在石头和深板岩中生成
- **铝土矿**: 矿脉大小7，主要在丛林生物群系
- **铜镍矿**: 矿脉大小6，与铜矿分布相似

### ModPlacedFeatures.java

定义矿石的放置特征，包括生成高度和频率：

```java
public static void bootstrap(Registerable<PlacedFeature> context) {
    // 石膏矿石放置 - Y=30到Y=80，每个区块8个矿脉
    register(context, GYPSUM_ORE_PLACED_KEY, 
            configuredFeatureRegistryEntryLookup.getOrThrow(ModConfiguredFeatures.GYPSUM_ORE_KEY),
            ModOrePlacement.modifiersWithCount(8, 
                    HeightRangePlacementModifier.uniform(YOffset.fixed(30), YOffset.fixed(80))));
}
```

**生成参数**：
- **石膏矿石**: Y=30-80，每区块8个矿脉
- **铝土矿**: Y=0-64，每区块6个矿脉
- **铜镍矿**: Y=-16-112，每区块4个矿脉

### ModOrePlacement.java

提供矿石放置的辅助方法：

```java
public static List<PlacementModifier> modifiersWithCount(int count, PlacementModifier heightModifier) {
    return List.of(
        CountPlacementModifier.of(count),    // 每个区块的生成次数
        SquarePlacementModifier.of(),        // 在区块内随机分布
        heightModifier                       // 高度范围限制
    );
}
```

**放置修饰符**：
- **计数修饰符**: 控制每个区块的生成次数
- **方形修饰符**: 在区块内随机分布
- **高度修饰符**: 限制生成的高度范围

### ModWorldGeneration.java

将矿石生成特征注册到生物群系中：

```java
public static void generateModWorldGen() {
    // 石膏矿石 - 在所有主世界生物群系中生成
    BiomeModifications.addFeature(BiomeSelectors.foundInOverworld(),
            GenerationStep.Feature.UNDERGROUND_ORES, ModPlacedFeatures.GYPSUM_ORE_PLACED_KEY);
    
    // 铝土矿 - 只在丛林和热带草原生物群系中生成
    BiomeModifications.addFeature(BiomeSelectors.includeByKey(
            BiomeKeys.JUNGLE, BiomeKeys.SPARSE_JUNGLE, BiomeKeys.BAMBOO_JUNGLE,
            BiomeKeys.SAVANNA, BiomeKeys.SAVANNA_PLATEAU, BiomeKeys.WINDSWEPT_SAVANNA),
            GenerationStep.Feature.UNDERGROUND_ORES, ModPlacedFeatures.BAUXITE_ORE_PLACED_KEY);
}
```

**生物群系分布**：
- **石膏矿石**: 所有主世界生物群系
- **铝土矿**: 丛林、稀疏丛林、竹林、热带草原及其变种
- **铜镍矿**: 所有主世界生物群系

## Mixin系统

Mixin是Fabric模组开发中用于修改原版游戏代码的强大工具。本模组使用了几个Mixin来实现特定功能。

### realschool.mixins.json

Mixin配置文件定义了所有的Mixin类：

```json
{
  "required": true,
  "minVersion": "0.8",
  "package": "papercliper.realschool.mixin",
  "compatibilityLevel": "JAVA_21",
  "refmap": "realschool.refmap.json",
  "mixins": [
    "AbstractFurnaceBlockEntityMixin",
    "EndermanEntityMixin",
    "ExampleMixin",
    "GrassColorsMixin"
  ],
  "client": [],
  "injectors": {
    "defaultRequire": 1
  }
}
```

### AbstractFurnaceBlockEntityMixin.java

修改熔炉方块实体的行为，可能用于：
- 自定义燃料系统
- 修改烧炼时间
- 添加特殊配方支持

### EndermanEntityMixin.java

修改末影人实体的行为：
- 可能限制末影人搬运某些模组方块
- 添加与模组内容的特殊交互

### GrassColorsMixin.java

修改草地颜色系统：
- 自定义草地颜色映射
- 可能用于特殊生物群系的视觉效果

### ExampleMixin.java

示例Mixin，展示基本的Mixin使用方法：
- 用于学习和测试Mixin功能
- 可以作为其他Mixin的模板

### 代码规范

#### 命名规范
- **类名**: 使用PascalCase，如`ModItems`、`StudentEntity`
- **方法名**: 使用camelCase，如`registerModItems()`
- **常量**: 使用UPPER_SNAKE_CASE，如`CHALK_BLOCK`
- **包名**: 使用小写，如`papercliper.realschool.item`

#### 注释规范
```java
/**
 * 物品注册类
 * 负责注册所有模组物品到游戏中
 */
public class ModItems {
    // 粉笔 - 用于在黑板上书写
    public static final Item CHALK = registerItems("chalk", new Item(new Item.Settings()));
    
    /**
     * 注册物品到游戏注册表
     * @param id 物品ID
     * @param item 物品实例
     * @return 注册后的物品
     */
    private static Item registerItems(String id, Item item) {
        return Registry.register(Registries.ITEM, Identifier.of(RealSchool.MOD_ID, id), item);
    }
}
```

- 每个功能模块使用独立的包
- 相关的类放在同一个包中
- 数据生成类统一放在`datagen`包中
- 自定义物品类放在`item.custom`包中

## 已知问题和解决方案

### 1. 矿石生成问题
**问题**: 矿石没有按照规则生成在指定层数
**解决方案**: 
- 已修复世界生成代码
- 添加了正确的配置特征和放置特征
- 确保在主模组类中正确注册世界生成

### 2. 数据生成问题
**问题**: 中英文名称没有正确显示，配方可能不工作
**解决方案**:
- 检查语言文件生成器
- 确保数据生成任务正确执行
- 验证所有翻译键的正确性

### 3. 果汁逻辑问题
**问题**: 果汁使用了食物逻辑而不是药水逻辑
**解决方案**:
- 创建了自定义的`JuiceItem`类
- 实现了药水机制，给予随机正面效果15秒
- 使用`UseAction.DRINK`动作和相应的音效

### 4. 材质文件缺失
**问题**: 所有材质显示为错误材质
**解决方案**:
- 需要手动添加材质文件到对应目录
- 确保材质文件命名与物品/方块ID一致
- 材质文件应为16x16像素的PNG格式

### 5. GUI无法打开
**问题**: 功能方块的GUI无法打开
**解决方案**:
- 需要实现对应的`BlockEntity`类
- 添加GUI处理逻辑
- 注册屏幕处理器
